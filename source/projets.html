Jitsi

Jitsi est un téléphone Internet audio/video ainsi qu'un outil de messagerie instantanée qui supporte des protocoles de téléhonie et de messagerie instantanée parmis les plus populaires, tels que SIP, Jabber, AIM/ICQ, MSN, Yahoo! Messenger, Bonjour, IRC, RSS et bien d'autres.

Jitsi est un logiciel libre (Open Source / Free Software) et est disponible sous les termes de la licence "GNU Lesser General Public License".

dict4j

dict4j est une librarie Java permettant de se connecter avec un serveur de dictionaire au format DICT (RFC 2229). Elle a été développée en collaboration avec Cédric Litzelmann lors d'un projet de première année de Master d'Informatique pour SIP-Communicator.

Le code source est disponible sur java.net à l'adresse suivante : https://dict4j.dev.java.net/ (dépôt SVN).

phpSysInfo

phpSysInfo est une interface web écrite en php permettant de récupérer du serveur sur lequel il est installé et de les présenter à l'utilisateur.

Site web du projet : http://phpsysinfo.sourceforge.net/.

Projet SensLAB

L'objectif du projet SensLAB est le déploiement à grande échelle d'une plateforme ouverte de réseau de capteurs sans-fil. Le principal but, et le plus important, est d'offrir un outil scientifique efficace et précis pour aider à la construction, développement, amélioration et expérimentation d'applications réelles pour de vastes réseaux de capteurs sans-fil.

Cette plateforme est déployée dans 4 villes de France : Grenoble, Lille, Rennes, Strasbourg. Pour plus d'informations :

  • Site Web du projet
  • Site Web de la plateforme de Strasbourg

Contributions

  • Participation au développement d'une interface graphique de visualisation et de contrôle (interne à l'équipe).
  • Participation à la conception de démos utilisées lors de nombreux événements

WSNet

WSNet est un simulateur de réseaux sans fil développé par l'INRIA de Lyon. Chaque nœud est conçus comme un empilement blocs (librairies écrites en C) qui peuvent représenter soit un équipement physique (carte radio), une application ou une ressource.

Pour plus d'informations consultez le site du projet.

Contribution

  • Correction d'un bug dans la couche MAC idéale créant une erreur de segmentation lors du routage sur des grandes topologies. Amélioration de cette même couche pour prendre en compte le temps de transmission lors de l'envoi d'un paquet.
